03

Event Pipeline — From Detection to Directive

🔄
Step 01
10 Hz Simulation Tick
The asyncio event loop fires every 100 ms. Each tick advances drone positions via sector sweep, applies battery drain (base decay + payload coefficient), recomputes the dynamic RTH threshold from Euclidean distance to the launch station, and checks for thermal detections.
async def simulation_loop() -> None: while True: tick_start = loop.time() simulation_tick(state) # advance all drones await manager.broadcast(state.to_broadcast_payload()) await asyncio.sleep(max(0.0, TICK_INTERVAL - elapsed))
🔋
Step 02
Dynamic Battery Decay
Battery drains at the base rate every tick. When a drone is carrying its first-aid payload, an additional payload weight coefficient is applied — modelling increased motor load. RTH is triggered the instant battery falls to or below the dynamically calculated threshold.
# Base + payload weight coefficient payload_drain = BATTERY_PAYLOAD_COEFFICIENT if drone.payload_ready else 0.0 drone.battery = max(0.0, drone.battery - BATTERY_DECAY_PER_TICK - payload_drain) # Dynamic RTH threshold — distance-vector formula dist_to_base = _distance(drone.x_coord, drone.y_coord, base.x, base.y) drone.required_return_battery = dist_to_base * 0.05 + 5.0
⚡
Step 03
APF Collision Avoidance
After all drones tick, an Artificial Potential Field repulsion pass runs. A unit vector is computed between drones and a repulsion force proportional to the overlap ratio displaces both units apart, clamped to their sector boundaries.
# Compute unit repulsion vector ux = (beta.x_coord - alpha.x_coord) / dist uy = (beta.y_coord - alpha.y_coord) / dist repulse = (overlap / COLLISION_RADIUS) * DRONE_SPEED * 0.5 alpha.x_coord -= ux * repulse # push alpha away beta.x_coord += ux * repulse # push beta away
🚨
Step 04
Thermal Detection → SURVIVOR_FOUND Trigger
When a drone enters the 10 m detection radius of an unfound thermal signature, it marks the survivor found, deploys its payload, and enqueues a structured event to the async LLM queue — without blocking the simulation loop.
# Detection — enqueue event, never block sim loop event = { "drone_id": drone.drone_id, "x": thermal.x, "y": thermal.y, "battery": drone.battery, "situation": "Heat sig near collapsed structure…" } llm_event_queue.put_nowait(event)
🤖
Step 05
AEGIS-COMMANDER — LLM Tactical Directive
A background coroutine drains the event queue. For each survivor event it calls Groq llama-3.3-70b-versatile with a strict system prompt enforcing exactly 2 sentences: a drone directive and a ground team rendezvous order. The response is injected into the mission log and broadcast immediately.
# System prompt enforces structured 2-sentence output """ Sentence 1 → "Priority: HIGH/CRITICAL. [drone action + coords]" Sentence 2 → "Ground Command: [medical team + callsign rendezvous]" """ decision = await get_llm_tactical_decision(**event) state.log(f"🤖 [AEGIS-COMMANDER] {decision}") await manager.broadcast(state.to_broadcast_payload())
📡
Step 06
ESP32 Mock Bridge — ESP-NOW Mesh Packets
A parallel coroutine simulates the ESP32 base station transmitting ESP-NOW telemetry frames at 1 Hz. Each burst emits per-node packets with realistic RSSI, sequence numbers, and the full drone state — drop-in replaceable with a real pyserial port in the field.
packet = { "node_id": drone.drone_id, "rssi_dbm": random.randint(-80, -45), "packet_seq": packet_seq, "x": drone.x_coord, "battery": drone.battery } state.log(f"📡 [ESP32-RX] SEQ#{packet_seq:05d} | …")
🖥️
Step 07
WebSocket → Live Dashboard
Every tick broadcasts a JSON frame to all authenticated WebSocket clients. The Next.js dashboard renders the radar grid, drone telemetry cards, mission log, and launch station control in real time. On close code 1008 (invalid token) the frontend auto-clears the session and returns to the login screen.
// Frontend — JWT-gated WebSocket connection const url = `${WS_BASE}/ws/telemetry?token=${encodeURIComponent(jwt)}`; const ws = new WebSocket(url); ws.onmessage = (e) => setFrame(JSON.parse(e.data)); // 1008 = token rejected → force logout if (event.code === 1008) { sessionStorage.removeItem("aegis_jwt"); }

06

Security Hardening

✓
JWT Bearer Tokens
HS256 tokens issued by POST /token. 60-minute expiry. Signed with a server-side secret loaded from environment variables.
✓
bcrypt Password Hashing
passlib CryptContext with bcrypt scheme. Passwords are hashed at startup — never stored in plaintext. Timing-safe verify() used for comparison.
✓
WebSocket Auth Gate
Token validated before ws.accept() is ever called. Invalid or missing token → close code 1008 Policy Violation. No data leaks on unauthenticated connections.
✓
Session-Scoped Storage
JWT stored in sessionStorage (not localStorage). Token is cleared on tab close, explicit logout, or 1008 server rejection. No persistent credential exposure.
✓
Secure Login Modal
Full-screen overlay gating the entire dashboard. POSTs credentials as application/x-www-form-urlencoded to match FastAPI's OAuth2PasswordRequestForm. Error messages do not leak whether user or password was wrong.
✓
Render.com Auto-Secret
render.yaml uses generateValue: true for AEGIS_SECRET_KEY so each deployment gets a unique cryptographic secret. GROQ_API_KEY is environment-injected — never in source.
